Agamemnon

Live token spend from the menu bar.

You run three CLIs. Agamemnon watches them all: Kimi Code, Cursor agent, and Claude Code (claude-work). Input, output, cache, burn rate, rolling limits, and estimated cost. Nothing leaves your machine.

git clone anishfyi/agamemnon ./build.sh
Priority 1

Kimi Code CLI

Priority 2

Cursor CLI

Activity always; tokens when locally available

Priority 3

Claude Code (work)

Transcripts at ~/.claude-work/projects/

dashboard

What you see

One real-time panel per source, plus a totals strip across the top.

Menu-bar helmet with today's total tokens. Turns warning-colored when abuse alerts fire.
5-hour and weekly windows as progress bars with configurable per-source limits. Warning at 70%, critical at 90%.
Burn rate in tokens/min over the last 15 minutes, plus estimated USD cost from an editable pricing table.
Sessions and abuse rules: burn spike, daily cap, cache-miss anomaly, loop detection. All thresholds configurable.
Incremental parsers with a local SQLite cache. Restarts are cheap. Migrates from Warden on first run.
install

Build and run

macOS 13+, Swift 5.9+, Xcode Command Line Tools. Unsigned personal-tool distribution.

git clone https://github.com/anishfyi/agamemnon.git
cd agamemnon
./build.sh
open Agamemnon.app

If Gatekeeper blocks the app: xattr -cr Agamemnon.app

privacy

On disk only

No network access. No telemetry. Data lives at ~/Library/Application Support/Agamemnon/agamemnon.db.
